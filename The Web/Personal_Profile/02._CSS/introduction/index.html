Order of Execution

  • CSS prioritize ID's over class or element type styling.
  • That is to say, if you had styled your paragraph in a section of HTML to be red, and then styled font color of the section to be black by using a section ID, your paragraph will be black.
  • So when styling, start by targeting the element type first. This will apply the styling to all similar elements of course, but you can go after the ones you specifically want by assigning classes to them or assigning ID's

Intermediate CSS

Properties

  • These are properties of the HTML that you'll be styling (font-families, font-size, font-color, etc)

Font

  • You can style the typeface by using the following properties:
    • font-family: for the type of typeface you want
    • font-size: for the size. Make sure to add the "px" at the end of the font size you just entered. Eg p {font-size: 18px;}
    • text-align: for text alignment. It accepts left, center, right, and justify as arguments. Eg.
    • color: This changes the color of the text itself. background-color changes the color of the text box behind the text.
    • opacity: This controls the opacity of the text-box. Use this if you want to create a beautiful overlay effect
    • background-image is able to change the background of the text-box to be a picture.
    • The (!important) is used to prioritize the styling of any property it is put upon, no matter how much specifically the element might have been modified by another selector. It is rarely used, so you should also develop a habit of avoiding it. Eg

The Box Model

  • This is the way the elements are organized in an HTML. Each element is inside a box, and gets centered or expanded depending on the size and configurations made of the box.
  • As you can see from the picture, padding is the spacing between the actual content of the element and the borders of the box enclosing it.
  • The margin is the relationship between the box and the sides of the page.

The Content Properties

  • width and height modifies the size of the content within the box in pixels.
  • padding is the amount of space in pixels btn the content and the border
  • border modifies the width of an element's box borders in pixels
  • margin sets the margin for the box, also in pixels.

Properties of Margin

  • Margin has the same properties as padding. Even the short-handing is also the same.
  • One of the things to keep in mind is that, when you have two elements on top of each other, for example, and one of them has a margin of 20px while the other 30px, the browser will use the bigger margin to separate the two elements. This is known as margin collapse. It doesn't happen if you set equal margins to the two elements though. In such a case, the margin will be the total of the two margins of the elements.

General Box Rules

  • You can set the min-height, min-width rulesets to control how narrow or wide an element's box can be stretched or compressed. The height and width will include the margins of an element.
  • Setting it this way helps regulate how much a content will be stretched or compressed under different screen sizes and still be legible. When it gets too compressed, it usually "spills" out of its box, necessiating the "overflow" property.   • The "visibility" property hides or displays an element on the page.

A Better Box

  • The box outlined above suffers from the fact that any setting on padding, margin, and border, affects the overall width and height of the box. This makes it difficult to accurately program the size of the elements in your page.
  • To solve this problem, use the box-sizing property with the universal (*) selector at the beginning of your CSS and set it to (border-box)
  • The reason the default box doesn't do what "border-box" does is because the default CSS value of all elements is "content-box". So, when the boxes are in content-box mode, the values of height and width you specify are applied to the contents of the box, not the box itself.
