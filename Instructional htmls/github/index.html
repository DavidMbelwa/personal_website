Git

The Main Places Git Operates

The Working Directory

  • This is the folder in your computer where all the current files of a project are stored
  • The first step in any version control tracking involves identifying this folder and set

The Staging Area

  • This is only known to git itself, and it is a place where changes made are temporarly recorded so that they can be latter permanently saved or discarded.
  • Only files that have been added and that are tracked by Git can be able to be viewed in their altered state in the staging area. Apart from that, the changes made to the file aren't being tracked.
  • Depending on where you're writing your code, staging can simply mean clicking the "Save" button in your IDE (VS Code for example), to a deliberate staging (in RStudio for example). Ask the internet concerning your particular code editor.

The Repository

  • This is an online copy of the working directory. Its main purpose apart from sharing your code files is to be able to display the changes made to a tracked file, both locally and remotely, so that who ever is reading/following your project understands how you've modified the project overtime.
  • Repositories stay in online platforms such as GitHub, Beanstalk, and others. GitHub is the current most popular.

The Git Workflow

  • Initiate a git repository by using git init
  • Use git add file_name to add a file to the staging area of Git.
  • Check the changes/differences btn the file in the staging area and the same file in the working directory by using git diff file_name.
  • Add the changed file from the working directory to the staging area of Git. Do this by using git add file_name. This will basically update the content of the document in the staging area.
  • Commit the changes made on the file by using git commit -m "A short line explaining what you changed (the commit message.)"
  • Use git log to read commit messages in chronological order and view how the project evolved overtime.
  • git status is for checking which files are untracked in the repository, and which files were changed of the tracked ones.
