Before You Start:

  • Always design for mobile first.
  • In the same spirit, set your columns first (their width) and then specify how many columns each element will span in the HTML. Do this for every major section of your document. Color

  • In CSS, you can use either the named colors, the hexadecimal colors and rgb colors to create a color out of a mixture of red, blue and green, or use the HSL colors to create colors by controlling their hue, saturation and lightness.
  • All colors among named colors have their respective hexadecimal and rgb values, but since the hexadecimal and rgb values are hard to remember, people who started this business earlier than you decided that it is wise to name a certain set of commonly used colors in order to avoid the hustle of trying to record each number or letter that is in the hexadecimal or RGB values.
  • Hecadecimal and RGB on the other hand, allow you to explore a wide range of colors, simply by changing the respective values of red, green, and blue. So these two options are greater for exploration.
  • HSL is among the best. With it, you not only control how green, blue, or red a color is, but also how saturated (rich or pale), and how light a color is.

  • This is the way the elements are organized in an HTML. Each element is inside a box, and gets centered or expanded depending on the size and configurations made of the box.
  • As you can see from the picture, padding is the spacing between the actual content of the element and the borders of the box enclosing it.
  • The margin is the relationship between the box and the sides of the page.

  • border modifies the width of an element's box borders in pixels
  • margin sets the margin for the box, also in pixels.

Box Layout

  • Any HTML element is either a block or an inline element.
  • Block elements are the ones that takes the whole width of the screen i.e., they span the whole horizontal space of the part of the page they are located at. Inline elements share the horizontal space with other elements in the HTML.
  • Elements such as headings, tables, and paragraphs, are by default block elements, while elements such as strong, em and the like are by default inline elements.
  • The display property can be used to make a block element an inline one and vice versa.
  • In the code above, the p element will share the horizontal space with the img element, displaying them side by side in the browser.
  • display: inline; however, doesn't allow you to modify the width and height of the element. To achieve that, you use display: inline-block;
  • inline-block allows you to make a block element an inline, but formats the element such that it can be modified as if it is a block element.
  • You can then modify the height and width, not only padding, margin, and border as it is for an inline ruleset.
Box Float
  • This is a layout property used to push an HTML element's box as far left or right as possible.
  • It takes only two arguments: left and right.
  • The width of the element must be specified however. If it hasn't, the floated element will assume the whole width of its containing element.
"Clear"
  • This is used to control the behaviour of floated elements (most of the time) by specify how an element should behave in a situation where it is going to bump into another element horizontally.
  • Bumping can easly happen with the float property, but if you add the clear property to any of the floated element and set it to either left, or right, or both, that element won't touch any other element it finds to its left, right or both sides.

The Flex Box

  • It is a styling system that makes it possible for HTML to resize based on the size of the screen.
  • It is set by setting the display property of an element to either "flex" or "inline-flex". This will make the element a flex container
  • The flex container has properties that enable items within it to resize and re-position themselves based on how stretched the box is.
  • display: flex; will make the container a block element while the display: inline-flex; will make it an inline-block.
  • Once that is done; you can go inside and start commanding how the elements within the container should behave. The first property is justify-content.

Properties of Flex Containers

Justify-content
  • It is a property that is declared on the container in order to align the items within the container horizontally. It can take any of the following values:
    • flex-start: Which will align everything in the flex container to the left.
    • flex-end: Which will align all items in the flex container to the right.
    • center: As is self-explanatory, will center the content of a flex container
    • space-around: This will put equal space on the sides (left-right) of each element, with regard to margins also.
    • space-between: This will put equal space between items themselves, but no space at the right side of the left-most item and the left-side of the right-most item (the last item)
  • With justify-content, you're basically instructing the browser to stuck together the items within the flex container in an inline fashion in the event the screen is squeezed in. Read further down to learn how you can stretch the items vertically or horizontally.
"align-items"
  • align-items: is another property applied to the container, and it aligns the items vertically and takes the following rulesets:
    • flex-start: which aligns the items on the top of the container.
    • flex-end: which aligns the items on the bottom.
    • center: centers, as it says.
    • baseline: which aligns the bottoms of each element
    • stretch: which stretches the items the whole height of the container (will not go beyond the max-height instructions though.).

"flex-direction"
  • The flex property operates in two major axes: The horizontal and The vertical
  • The horizontal axis: flex-direction: row; is a default value of the flex property.
  • flex-direction: row; Tells the browser to put the items inside the flex-container into a horizontal axis i.e. the row, and then accepts modifications such as justify-content and others we saw above, as well as the ones applicable to the flex-items (Read below)
  • In flex-direction: column;, you basically instruct the browser to put the items of a flex container into a vertical alignment i.e. a column, instead of a row. From there on, you can use the same rulesets you use on flex-direction: row;, just that everything will behave as if you rotated your screen 90 degrees.
  • Values accepted for flex-direction are:
    • row: Which will just put the elements in the container is a single row, from left to right, starting from the top-left.
    • row-reverse: Where the items will be positioned from right to left, from the top-right corner.
    • column: Which will behave exactly like row, but put the items in a vertical column. Even the positioning starts from the top-left corner of the parent element.
    • column-reverse: Which will do what row-reverse does, just with the item vertically oriented.

Centering The Content within the Grid

On the horizontal Axis
  • Done with the combination of justify-content: ; and justify-items: ; rulesets
  • justify-items align items within the columns, that is, whether the item is on the left of the column, right, centered etc. It takes values of start for a left alignment, end for a right alignment, center for a centered alignment, and stretch for the item to stretch the whole width of the column.
  • justify-content aligns the whole of the grid horizontally within the grid container itself.
  • Assume a grid container had the width of 1300px and 2 columns, each one 500px. The justify-content: ; ruleset will instruct the browser to either align to the left, right or center, the 2 columns of the grid, such that the remaining 300px are used as the designer intended.
  • justify-content: ; accepts start, end, center, stretch, space-around, space-between, and space-evenly as arguments.
On the vertical axis
  • A combination of align-items and align-content is used.
  • The usage is the same as justify-items / justify-content, and even the arguments are the same. So use them as you would use those, except everything is done row-wise rather than column wise.
Justify-self / align-self
  • These are values declared on grid items, rather than the container, to instruct a particular item to deviate away from the styling declared on the container using justify-items / align-items
  • They accept the same values as justify-items and align-items, just that their customization applies to just one grid item on which they're declared.

Grid-auto-flow

  • This is a property set on a grid container when the developer doesn't know exactly how many rows / columns the grid will have.
  • Take an example of a grid that comes as a result of a search inquiry on a site. Sometimes, these grids have an option of how many search results one wants displayed at a time, making it impossible for the developer to guess how many items will be in that grid in a particular point in time. To deal with this, the grid-auto-flow ruleset is used to specify whether the additional content that is more that what is specified in the original grid is placed on new columns or rows automatically.
  • By the way, the grid we have seen so far is called the explicit grid, because it has been explicity designed and structured in terms of the number of rows / columns and their sizes.
  • The additional rows / columns that are added by grid-auto-flow are known as the implicit grid. The width and height of the columns and rows of these grids fit the width / height of the content by default, but can be specified by using the grid-auto-rows / grid-auto-columns rulesets. More on them in a bit.
  • grid-auto-flow accepts row, column, dense properties as values. row instructs the browser to add the new content to a new row, column to a new column, and dense instructs the browser to fill any empty space that was there in the grid earlier if the added content is of small size.

Grid-Auto-Rows / Grid-Auto-Columns

  • grid-auto-rows specifies the width of new rows that were added implicitly. By default, new rows are added automatically when the grid-auto-rows / grid-auto-columns even when the grid-auto-flow hasn't been set.
  • To add the new content into columns, one has to explicitly specify by using grid-auto-flow: column; that the new content will go into columns. After doing that, they can move on specifying the width of those columns by using the grid-auto-columns. It is better to assign this the same width as the columns assigned explicitly using grid-template-columns.
