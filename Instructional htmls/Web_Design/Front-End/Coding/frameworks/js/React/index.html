React JS

Hello! Welcome!

This is page contains introductory notes on react.

What is React JS

React is a JavaScript framework used to make interactive websites, especially Single Page Applications (SPAs). It is also quite popular. You won't go wrong learning it.


How to Create A React App

  • Download the latest node into your computer.
  • Navigate to where you want to open your new React app and open the terminal there.
  • In the terminal, type npx create-react-app name_of_the_app_goes_here
  • After everything downloads and sets up, open your code editor and navigate to the repository of your React app. There, you'll see various folders named: node_modules, public, and src, and other json files. You'll be mainly working on a file called App.js that is under the "src" folder. Locate it now if you can.
  • In the "src" there are some files that you won't need as a beginner. The file names are: reportWebVitals.js and setupTests.js. You have to delete them for now so that they don't confuse you. After deleting them, go into the index.js folder and delete the line "import reportWebVitals from './reportWebVitals';" and "reportWebVitals()". You're good to go after you do that.
  • There are some things you should know about and they're located inside the package.json file. Go inside there now.
  • Inside the file, find the word "scripts". It should be inside quotation marks. Inside it are some of the scripts that you'll use to carry out some functionalities with your React app. One of them that is important to you right now is the "start" script. It enables you to preview the changes you're making on your React app as you code, in real time. To see how it works, open a terminal in your working directory and type npm run start to execute it. Your React App should show up in your default browser in a few moments.

Initiating Your React App

  • Now that you can preview your app, you would automatically want to remove that React logo and everything else and start coding your own interface.
  • To do this, go to your App.js and delete the line import logo from './logo.svg'; as well as all the jsx code that is within the div with a class name "App". After you do that, you can go to the src folder and delete the logo.svg file that is there. You should now see a blank page in the preview window of your browser when you save the changes you just made.
  • To start creating on your React App, you're supposed to know about React components. Read the next section below.

React Components

  • React compoonents are essentially design components such as navbars, accordions, etc.
  • You usually code them in separate files and then import them inside the App.js's root function. Consider the example below.

React and Stylesheets

  • The React app comes natively with two major style files: App.css and index.css. They are located on the src folder.
  • To insert your own css, you can delete the App.css file from the src folder, delete its import line from App.js, and then put all your styling in the index.css file. You're then good to go.
  • If you're planning to use a css libray like Bootstrap, install that library in your working directory, hook it up to your project by importing its css on the index.js file, and then delete the index.css file from the src folder and its import on the index.js file after this.
  • For bootstrap, there are instructions on how to hook up bootstrap css on a project on the section about bootstrap.

The "node_modules" Folder

  • This is the folder that will contain all frameworks that your React app will depend on, even React itself.
  • It is usually quite large, so it usually doesn't get uploaded into version controll managers such as git and github. Instead, since all the dependencies in this folder are listed in the "package.json" file of your project, when you clone a project, let's say on Github, and it doesn't have the "node_modules" folder inside of it, the only thing you need to do is to type npm install in a terminal inside your working directory, and npm will read the "package.json" file and download all the required frameworks. You don't need to sweat over that.
